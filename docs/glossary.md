# Словарь терминов — Floating Spheres

Алфавитный словарь сложных и специализированных терминов, используемых в [документации анимации](/animation) и в коде проекта.

---

## A

### Axis-aligned bounding box (AABB)

**Ограничивающий прямоугольник, выровненный по осям.** Прямоугольная область, стороны которой параллельны осям координат $x$ и $y$. В проекте canvas — AABB, внутри которого должны оставаться сферы.

---

## C

### Canvas 2D API

**Набор методов браузера** для рисования на элементе `<canvas>` через контекст `CanvasRenderingContext2D` (`arc`, `stroke`, `translate` и т.д.).

### Continuous Collision Detection (CCD)

**Непрерывное обнаружение столкновений.** Метод, при котором симуляция вычисляет **точный момент** столкновения между дискретными кадрами, а не проверяет пересечение только в конце шага. Предотвращает эффект **tunneling** (проскальзывание сквозь объекты при большой скорости или Δt).

### Context (контекст рисования)

Объект `CanvasRenderingContext2D`, через который выполняются все команды отрисовки на canvas.

---

## D

### Delta time (Δt)

**Приращение времени** между двумя последовательными кадрами анимации, в секундах. Используется для независимости скорости симуляции от частоты кадров монитора.

### Design reference size (эталонный размер дизайна)

Базовый размер **800 px**, относительно которого задаются радиусы и скорости. При другом размере окна значения масштабируются пропорционально.

### Design units (дизайн-единицы)

Условные единицы измерения в «макете» 800×800 px. Например, `SPHERE.speed = 0.3` означает 0.3 px за кадр при 60 FPS в эталонном масштабе.

### Discriminant (дискримinant)

**Дискримinant квадратного уравнения** $D = b^2 - 4ac$. Если $D \lt  0$, действительных корней нет — сферы не соприкоснутся. Если $D \ge 0$, корни дают возможные моменты касания.

### Dot product (скalarное произведение)

Операция над двумя векторами: $\vec{a} \cdot \vec{b} = a_x b_x + a_y b_y$. Показывает, насколько векторы «сонаправлены». Используется для проекции скорости и проверки сближения объектов.

---

## E

### Elastic collision (упругое столкновение)

Столкновение, при котором **кинетическая энергия** (в полной физической модели) сохраняется. В проекте используется **упрощённая** упругая модель: отражение скорости от нормали без учёта масс и с принудительной фиксацией модуля скорости.

### Engine (движок)

Класс `Engine`, объединяющий рендерер, viewport и сцену и управляющий главным циклом анимации.

### Euclidean distance (евклидово расстояние)

Стандартное расстояние между двумя точками на плоскости: $d = \sqrt{(x_2-x_1)^2 + (y_2-y_1)^2}$.

### Event-driven simulation (событийная симуляция)

Подход к физике, при котором время продвигается **от события к событию** (столкновение, касание стены), а не фиксированными большими шагами.

---

## F

### FPS (Frames Per Second)

**Кадров в секунду.** Эталон проекта — 60 FPS; скорость в дизайн-единицах пересчитывается в px/s с умножением на 60.

---

## K

### Kinematics (кинematics)

Раздел механики об **движении без учёта сил**. Здесь — равномерное движение: $\vec{r}(t) = \vec{r}_0 + \vec{v}t$.

---

## N

### Normal vector (нормаль)

**Единичный вектор**, перpendicularный поверхности столкновения. Для двух сфер — направление от центра A к центру B: $\hat{n}_{A \to B}$. Для стены — ось, перpendicularная стене (например, $(1,0)$ для левой стены).

### Normalization (нормализация)

Приведение вектора к **единичной длине** (модуль = 1) с сохранением направления: $\hat{v} = \vec{v} / \|\vec{v}\|$. В коде — масштабирование до фиксированного модуля `speed`.

---

## O

### Overlap (перекрытие)

Ситуация, когда две окружности **геометрически пересекаются**: расстояние между центрами меньше суммы радиусов. Исправляется функцией `pushSpheresApart`.

---

## P

### Positional correction (позиционная коррекция)

Смещение объектов после столкновения, чтобы **устранить численное перекрытие** и добавить малый зазор `gapAfterContact`. Не меняет физику импульса, только координаты.

---

## R

### requestAnimationFrame

**Браузерный API** для вызова функции отрисовки перед следующей перерисовкой экрана. Синхронизирует анимацию с частотой обновления дисплея.

### Reflection (отражение)

Изменение направления вектора скорости при ударе о поверхность. Формула: $\vec{v}^{\prime} = \vec{v} - 2(\vec{v}\cdot\hat{n})\hat{n}$.

---

## S

### Scalar (скalar)

**Число** (величина без направления), в отличие от вектора. Например, модуль скорости $\|\vec{v}\|$ — скalar.

### Scene (сцена)

Класс `Scene`: контейнер всех сфер, методы `update` (физика) и `draw` (отрисовка).

### Sphere (сфера)

В 2D-контексте проекта — **окружность** с центром $(x,y)$, радиусом `radius` и вектором скорости $\vec{v}$. Визуально — белый контур без заливки.

---

## T

### Tunneling (туннелирование / проскальзывание)

Артеfact дискретной симуляции: объект за один шаг «перепрыгивает» через препятствие, потому что столкновение между кадрами не было обнаружено. CCD в проекте это устраняет.

---

## V

### Vec2

Тип данных `{ x: number; y: number }` — **двумерный вектор** в декартовых координатах.

### Velocity (скорость)

Вектор $\vec{v} = (v_x, v_y)$, описывающий направление и величину движения. Модуль в проекте после столкновений принудительно равен `speed`.

### Viewport (область просмотра)

Класс `Viewport`: ширина и высота canvas в пикселях плюс коэффициент `scale` для перевода дизайн-единиц в экранные пиксели.

### Vite

**Сборщик и dev-сервер** для frontend-проектов. В проекте компилирует TypeScript и запускает локальный сервер (`npm run dev`).

---

## W

### Wall bounce (отскок от стены)

Обработка выхода окружности за границу canvas: коррекция координаты и инверсия соответствующей компоненты скорости ($v_x \leftarrow -v_x$ или $v_y \leftarrow -v_y$).

---

## Символы

| Символ | Значение |
|---|---|
| $\vec{v}$ | Вектор скорости |
| $\hat{n}$ | Единичный вектор нормали |
| $\|\vec{a}\|$ | Длина (модуль) вектора |
| $\Delta t$ | Приращение времени между кадрами |
| $\varepsilon$ | Малая величина зазора (`gapAfterContact`) |
| $W, H$ | Ширина и высота canvas |
| $r, R$ | Радиус одной сферы / сумма радиусов двух сфер |
